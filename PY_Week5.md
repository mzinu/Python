## 1. 텍스트 파일에서 데이터를 읽고 쓰는 법

### 개념정리

#### 6.1.0 핵심 개념

- 개념
  - 데이터 로딩(data loading): 데이터를 읽어서 바로 다룰 수 있게 만드는 과정
  - 파싱(parsing): 텍스트 데이터를 읽어 표 형태 등으로 해석하는 과정
  - 판다스에서 가장 많이 쓰는 함수는 pd.read_csv()

- 자주 등장하는 읽기 함수
  - read_csv : CSV/구분자 파일 읽기
  - read_excel : 엑셀 파일 읽기
  - read_json : JSON 읽기
  - read_html : HTML 안의 테이블 읽기
  - read_xml : XML 읽기
  - read_sql : SQL 쿼리 결과 읽기
  - read_parquet, read_feather, read_pickle : 이진 형식 읽기

- read_csv()에서 꼭 알아야 할 옵션
  - 헤더와 열 이름
    - header=None : 헤더 없는 파일 처리
    - names=[...] : 직접 열 이름 지정
  - 인덱스 지정
    - index_col="열이름" 또는 index_col=번호
    - 여러 열을 인덱스로 쓰면 계층적 색인(MultiIndex) 가능
  - 구분자 지정
    - sep="," 기본
    - 공백 여러 개 같은 경우 sep="\s+"처럼 정규표현식 사용 가능
  - 불필요한 행 건너뛰기
    - skiprows=[0,2,3]
  - 결측치 처리
    - na_values=[...] : 특정 문자열을 결측치로 인식
    - keep_default_na=False : 기본 NA 문자열 해제
  - 날짜 처리
    - parse_dates=True 또는 특정 열 지정
  - 일부만 읽기
    - nrows=5 : 앞부분만 확인
    - chunksize=1000 : 나눠서 읽기

#### 6.1.1 텍스트 파일 조금씩 읽기

- 큰 파일은 한 번에 다 읽지 말고 부분만 읽거나 나눠서 읽는 게 중요함

- 방법
  - nrows : 처음 몇 줄만 읽기
  - chunksize : 여러 조각으로 나눠 읽기
  - 반환 객체는 TextFileReader
  - 반복문으로 각 chunk를 순회하면서 집계 가능

- ex: 큰 CSV를 1000줄씩 읽어서 특정 열의 빈도수 합산 가능

#### 6.1.2 텍스트 형식으로 저장하기

- 핵심 함수
  - DataFrame.to_csv()

- 자주 쓰는 옵션
  - sep="|" : 구분자 변경
  - na_rep="NULL" : 결측치 출력값 지정
  - index=False : 행 인덱스 저장 안 함
  - header=False : 열 이름 저장 안 함
  - columns=[...] : 일부 열만 저장

#### 6.1.3 다른 구분자 형식 다루기

- 판다스로 안 되는 특이한 CSV는 파이썬 내장 csv 모듈로 직접 처리할 수 있다.

- 주요 객체
  - csv.reader() : 읽기
  - csv.writer() : 쓰기
  - csv.Dialect : 구분자, 인용부호, 개행 규칙 정의

- 중요 옵션
  - delimiter
  - quotechar
  - quoting
  - skipinitialspace
  - escapechar

#### 6.1.4 JSON 데이터

- 표준 라이브러리
  - json.loads() : JSON 문자열 → 파이썬 객체
  - json.dumps() : 파이썬 객체 → JSON 문자열

- 판다스와 연결
  - 딕셔너리 리스트 → DataFrame(...)
  - pd.read_json() : JSON → DataFrame
  - to_json() : DataFrame → JSON

- to_json()의 주요 형태
  - 기본 형식: 열 중심 구조
  - orient="records" : 레코드(행) 중심 구조

#### 6.1.5 XML과 HTML, 웹 스크래핑

- HTML 테이블 읽기
  - pd.read_html() : HTML 문서 내 <table> 자동 파싱
  - 결과는 DataFrame 리스트

- XML 읽기
  - 방법 1: lxml.objectify
    - XML 노드를 직접 순회하면서 딕셔너리 생성
    - 복잡하지만 세밀한 제어 가능
  - 방법 2: pd.read_xml()
    - XML을 한 줄로 DataFrame으로 읽기 가능

- HTML/XML은 계층 구조라서 CSV보다 복잡하다
- 간단하면 read_html, read_xml
- 복잡하면 lxml 직접 사용

## 5. 누락된 데이터 처리하기

### 개념정리

- 누락값(결측치)은 실제 데이터에서 아주 흔하다.
- 판다스는 이를 쉽게 다루도록 설계되었다.

- 결측치 표현
  - NaN
  - None
  - 둘 다 판다스에서 NA로 인식 가능

- 확인 함수
  - isna() : 결측치면 True
  - notna() : 결측치 아니면 True

#### 7.1.1 누락된 데이터 골라내기

- dropna() : 결측치를 제거하는 함수
- Series에서 dropna() : NA 제외

- DataFrame에서
  - 기본: NA가 하나라도 있는 행 제거
  - how="all" : 전부 NA인 행/열만 제거
  - axis="columns" : 열 기준으로 제거
  - thresh=n : NA가 아닌 값이 n개 이상인 행/열만 유지

#### 7.1.2 누락된 데이터 골라내기

- fillna() : 결측치를 다른 값으로 채움

- 방식
  - fillna(0) : 0으로 채우기
  - fillna({열이름: 값}) : 열마다 다르게 채우기
  - fillna(method="ffill") : 앞 값으로 채우기
  - fillna(method="bfill") : 뒤 값으로 채우기
  - limit=2 : 최대 몇 개까지 채울지 제한
  - fillna(data.mean()) : 평균값으로 대체

## 6. 데이터 변형 

### 개념정리

#### 7.2.1 중복 제거

- 관련 메서드
  - duplicated() : 중복 여부 확인
  - drop_duplicates() : 중복 제거

- 옵션
  - subset=[...] : 특정 열 기준 중복 검사
  - keep="last" : 마지막 값 유지

#### 7.2.2 함수나 매핑을 이용한 데이터 변형

- 핵심 메서드
  - map()

- 활용
  - 딕셔너리 매핑
  - 사용자 정의 함수 적용

- 예시 개념
  - 음식명 → 원재료(동물)로 변환

#### 7.2.3 값 치환하기

-  핵심 메서드
  - replace()

- 활용
  - 특정 감싯값(-999, -1000 등)을 NaN으로 바꾸기
  - 여러 값을 한 번에 치환
  - 딕셔너리 방식으로 치환

#### 7.2.4 축 색인 이름 바꾸기

- 관련 메서드
  - index.map()
  - rename()

- 활용
  - 행 인덱스/열 이름 전체 변경
  - 일부만 선택적으로 변경
  - 함수 기반 변경 가능

#### 7.2.5 이산화

- 연속형 데이터를 구간으로 나누어 범주형 데이터로 변환하는 작업

- 주요 함수
  - pd.cut() : 사용자가 구간 기준 직접 지정
  - pd.qcut() : 분위수 기준으로 비슷한 개수의 그룹 생성

- cut() 특징
  - 구간 경계 직접 설정
  - labels=로 이름 부여 가능
  - right=False로 포함 방향 변경 가능

- qcut() 특징
  - 사분위수, 분위수 기반 분할
  - 각 구간의 데이터 수가 비교적 균등

#### 7.2.6 이상치 찾고 제외하기

- 극단값(outlier)은 평균, 분산, 모델 성능에 큰 영향을 준다.

- 방법
  - abs() > 기준값
  - (df.abs() > 3).any(axis="columns") 로 이상치 포함 행 찾기
  - np.sign(data) * 3 같은 방식으로 상한/하한 자르기

#### 7.2.7 뒤섞기와 임의 샘플링

- 관련 함수
  - np.random.permutation() : 순서 섞기
  - take() / iloc[] : 섞은 순서 적용
  - sample() : 임의 표본 추출

- 옵션
  - n=3 : 3개 추출
  - replace=True : 복원추출 허용

#### 7.2.8 표시자, 더미 변수 계산하기

- 범주형 데이터를 숫자형 입력으로 바꾸기 위해 더미 변수(dummy variable) 를 만든다.

- 핵심 함수
  - pd.get_dummies()

- 활용
  - 범주형 열 하나를 여러 개의 0/1 열로 변환
  - prefix= 로 접두어 추가
  -  원래 데이터와 join()으로 결합

- 다중 범주 처리
  - 문자열 여러 개가 구분자로 들어간 경우
  - str.get_dummies() 사용
